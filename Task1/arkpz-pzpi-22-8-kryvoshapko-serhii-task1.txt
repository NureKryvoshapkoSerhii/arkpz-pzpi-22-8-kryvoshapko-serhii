1	Business Requirements (Бізнес-вимоги)
1.1	Background (Передумови)
У сучасному світі зростає потреба в інструментах, що допомагають людям слідкувати за своїм здоров’ям, підтримувати належний рівень фізичної форми та контролювати харчування. Однією з найбільш актуальних проблем є надмірне або незбалансоване споживання калорій та макронутрієнтів (білків, жирів і вуглеводів), що часто призводить до порушень обміну речовин, надлишкової ваги або недостатнього споживання необхідних поживних речовин. При цьому багато людей не мають достатньої інформації та знань для самостійного контролю харчування, що ускладнює досягнення особистих цілей, таких як схуднення, підтримка ваги або набір м’язової маси.
Існує низка додатків для підрахунку калорій, які дозволяють користувачам вводити спожиті продукти і контролювати енергетичну цінність їжі. Однак, більшість із них обмежується лише базовими функціями, такими як підрахунок калорій без детальної індивідуальної настройки або можливості коригування рекомендованої норми залежно від рівня фізичної активності та змін у показниках здоров'я.
Зважаючи на ці недоліки, програмна система, яка дозволяє користувачам легко відстежувати щоденний калорійний баланс та макронутрієнти, а також адаптує рекомендації відповідно до їх індивідуальних параметрів (вік, вага, зріст, цілі) та даних про фізичну активність, може суттєво покращити якість контролю харчування. Така система сприятиме підвищенню мотивації користувачів і забезпечить комфортний та ефективний інструмент для організації здорового способу життя.

1.2	Business Opportunity (Бізнес-можливості)
У сучасному світі, де люди все більше усвідомлюють важливість здорового способу життя, ринок програм для контролю калорій і харчування зростає динамічно. Існує багато додатків, що дозволяють користувачам рахувати калорії та відстежувати макронутрієнти, як-от «MyFitnessPal» та «Lifesum». Ці платформи пропонують базові можливості, такі як введення продуктів харчування, підрахунок калорій та базовий аналіз макронутрієнтів. Однак їхній функціонал часто обмежений, адже ці програми не завжди пропонують детальну персоналізацію, не адаптуються до змін у вазі користувача, та не мають можливостей для інтеграції з IoT-пристроями, які могли б автоматично зчитувати дані про вагу та відповідно коригувати калорійні рекомендації.
Більшість програм також не підтримують гнучкого підходу до різних цілей користувача, таких як схуднення, набір м'язової маси чи підтримка ваги. Користувачі потребують не лише обліку калорій, але й додаткових можливостей, таких як моніторинг прогресу, інтеграція з інтелектуальними пристроями для автоматичного зчитування ваги та детальне налаштування рекомендацій. Ці фактори ускладнюють використання наявних рішень для ефективного контролю здоров’я та харчування.

Програмна система для підрахунку калорій та контролю харчування вирізнятиметься серед конкурентів, пропонуючи комплексний підхід, що включає персоналізовані рекомендації на основі даних про вік, вагу, зріст, цілі та інші параметри користувача. Вона інтегруватиметься з інтелектуальними вагами для автоматичного обліку ваги, що дозволить коригувати денну норму калорій залежно від змін у вазі. Крім того, система матиме можливість відстеження прогресу користувача у досягненні поставлених цілей, надаючи йому мотиваційні дані та візуалізації, які підтримуватимуть зацікавленість та контроль процесу.
Ринок додатків для здорового способу життя та харчування демонструє стабільне зростання. Згідно з прогнозами, ринок застосунків для здоров’я й фітнесу буде зростати щорічно на 23% до 2030 року, що створює перспективи для нових рішень з інноваційними функціями. Інтеграція IoT-технологій у такі додатки має великий потенціал, оскільки автоматизація обліку ваги й харчування може стати новим стандартом у сфері здоров’я.
Модель монетизації для цієї системи передбачає кілька напрямків. Базовий доступ до платформи може бути безкоштовним, а платні підписки відкриватимуть доступ до розширених функцій, таких як кастомізація харчового плану, автоматичне оновлення цілей на основі змін ваги, інтеграція з інтелектуальними вагами, музика для тренувань тощо. Додатковий дохід можливий через партнерства з виробниками інтелектуальних ваг та фітнес-товарів, що забезпечить прибуток від продажів цих продуктів та їх інтеграції з платформою.



	
	 


2	Vision of the Solution (Концепція рішення)
2.1	Vision Statement (Окреслення концепції)

Програмна система для підрахунку калорій та контролю харчування спрямована на підтримку користувачів у досягненні їхніх цілей щодо здоров'я та харчування. Система допомагає користувачам оптимізувати споживання калорій та макронутрієнтів (білків, жирів, вуглеводів) відповідно до їхніх особистих параметрів (вік, вага, зріст, рівень фізичної активності) та цілей (схуднення, набір маси, підтримка ваги). Головною метою системи є надання індивідуальних рекомендацій, які забезпечать збалансоване харчування та підвищать ефективність користувачів у досягненні фізичних цілей.

Інтеграція з IoT-пристроями, такими як інтелектуальні ваги, дозволяє автоматично зчитувати дані про вагу користувачів та коригувати рекомендовану норму калорій, адаптуючи її до змін у вазі. Такий підхід забезпечує зручне та ефективне середовище для контролю власного прогресу, допомагаючи користувачам не лише досягати своїх цілей, але й підтримувати мотивацію на шляху до здорового способу життя.

2.2	Major Features (Головна функціональність)

Програмна система має кілька ключових функціональностей, спрямованих на облік індивідуальних цілей і дій користувачів. Система підтримує три основні ролі: користувачі (основні споживачі), фітнес-консультанти та адміністратори.

2.2.1	Функціональність для користувачів

MF-1. Реєстрація та авторизація

Користувачі можуть зареєструватися через Google, Facebook або інші соціальні мережі, що спрощує процес входу. Після авторизації обліковий запис користувача автоматично створюється у системі. Сервер обробляє дані автентифікації, зберігає базову інформацію, таку як email та ім'я, і надає доступ до особистого кабінету.

MF-2. Налаштування цілей харчування

Користувачі можуть задавати свої цілі щодо калорійності, білків, жирів і вуглеводів на день. На основі введених параметрів система розраховує оптимальні показники споживання. Усі дані зберігаються на сервері, що дозволяє користувачам стежити за своїм прогресом і переглядати аналітику досягнень.







MF-3. Автоматичне коригування на основі IoT-пристроїв

Система дозволяє користувачам підключати інтелектуальні ваги, які автоматично зчитують дані про вагу користувача щоразу після зважування. Інтелектуальні ваги передають інформацію через IoT-з’єднання на сервер, де вона обробляється та зберігається в базі даних. На основі отриманих даних система коригує індивідуальні рекомендації щодо споживання калорій, білків, жирів і вуглеводів, адаптуючи їх до змін у вазі користувача. Всі оновлення автоматично синхронізуються з профілем користувача, що дозволяє відстежувати прогрес і зберігати історію змін у вазі для аналізу.

MF-4. Ведення харчового щоденника

Користувачі можуть заносити дані про свої прийоми їжі, вводячи назву продукту або скануючи штрих-код, що автоматично заповнює дані про калорійність і макронутрієнти. Серверна частина обробляє інформацію, додає її до бази даних і оновлює загальні показники споживання за день.

MF-5. Перегляд аналітики та прогресу

Система збирає та зберігає дані про кількість спожитих калорій та макронутрієнтів, що дає змогу користувачам переглядати історію харчування за день, тиждень або місяць. 
Аналітичні звіти дозволяють порівнювати плановані показники з фактичними результатами, що допомагає користувачам покращувати планування свого харчування.

MF-6. Нагадування та сповіщення

Користувачі можуть налаштовувати сповіщення про прийоми їжі або питну воду. Система надсилатиме автоматичні нагадування, що допоможуть користувачам дотримуватися запланованого режиму харчування.

2.2.2 Функціональність для фітнес-консультантів

MF-7. Управління профілями користувачів

Фітнес-консультанти можуть мати доступ до профілів користувачів, які звернулися за допомогою. Вони можуть переглядати історію харчування, динаміку ваги, активність, що дає змогу коригувати плани харчування та тренувань відповідно до потреб користувачів.

MF-8. Надання рекомендацій

Фітнес-консультанти можуть залишати коментарі та рекомендації для користувачів у їх особистих кабінетах. Це дозволяє надавати персоналізовану підтримку, відслідковувати виконання цілей та допомагати користувачам у корекції харчування.

 

2.2.3 Функціональність для адміністратора

MF-9. Управління обліковими записами

Адміністратор має можливість управляти обліковими записами користувачів та консультантів, створювати або видаляти облікові записи, змінювати рівні доступу та стежити за активністю.

MF-10. Контроль контенту та даних

Адміністратор також може контролювати контент, завантажений у систему, включаючи навчальні матеріали та рекомендації, що дозволяє підтримувати якість контенту та відповідність стандартам платформи.

3	Scope and Limitations (Рамки та обмеження)
3.1	Scope of Initial Release (Рамки первинного запуску)

У першому випуску програмної системи для підрахунку калорій і контролю харчування будуть реалізовані основні функції серверної частини та інтеграція з IoT-пристроями для підтримки базових функцій обліку харчування, відстеження активності та управління щоденними цілями.

3.1.1	Серверна частина

Серверна частина виконує ключову роль у зберіганні та обробці даних користувачів, таких як інформація про харчові вподобання, рівень фізичної активності та індивідуальні параметри. Основна бізнес-логіка буде зосереджена на підтримці персоналізованих рекомендацій щодо щоденного споживання калорій та макронутрієнтів. Сервер отримуватиме дані від користувачів щодо спожитих продуктів, відстежуватиме прогрес, розраховує норму споживання калорії і генеруватиме звіти про досягнення цілей.

Серверна частина також забезпечить можливість авторизації, реєстрації та управління обліковими записами, надаючи користувачам різні рівні доступу, а також функціонал зберігання даних у режимі реального часу.

Крім того, система буде адаптована для кількох мов, таких як українська та англійська. Це забезпечить комфортну роботу для користувачів із різних країн.
 


3.1.2	IoT-клієнт

IoT-клієнт забезпечить інтеграцію з пристроями, такими як інтелектуальні ваги, для автоматизації обліку ваги користувачів. Після підключення вагів, дані про вагу автоматично передаватимуться на сервер щоразу після зважування. Отримані показники зберігаються у базі даних та використовуються для розрахунку коригованих рекомендацій щодо споживання калорій, білків, жирів і вуглеводів, адаптованих до змін у вазі користувача. Система автоматично оновлює профіль користувача, надаючи актуальну інформацію про прогрес у досягненні цілей та зберігаючи історію вимірювань для подальшого аналізу.

3.2	Scope of Subsequent Releases (Рамки наступних випусків)

У наступних випусках функціональність системи буде розширено для повної підтримки веб- та мобільних клієнтів, що забезпечить доступ до системи з будь-якого пристрою. 

3.2.1	Веб-клієнт

Веб-клієнт надасть доступ до платформи через браузер з повним функціоналом для адміністраторів. Адміністратори, матимуть доступ до наступних функцій через веб-клієнт.
Управління обліковими записами користувачів і консультантів: Адміністратори зможуть створювати, редагувати та видаляти облікові записи користувачів і фітнес-консультантів, а також надавати різні рівні доступу.
Перегляд звітів та аналітики: Адміністратори матимуть доступ до загальної аналітики використання платформи, статистики активності користувачів та взаємодії з IoT-пристроями, що дозволить їм оцінювати ефективність платформи та приймати рішення щодо її вдосконалення.

3.2.2	Мобільний клієнт

Мобільний клієнт буде розроблений для платформи Android і забезпечить зручний доступ до функцій платформи, включаючи відстеження харчування, активності, персоналізовані рекомендації та сповіщення про прийоми їжі. Користувачі зможуть використовувати мобільний додаток для введення даних про спожиті продукти через сканування штрих-кодів та переглядати свій прогрес у досягненні цілей.

Мобільний додаток також дозволить використовувати телефон як додатковий інструмент для відстеження активності, синхронізуючись з IoT-пристроями для автоматизації обліку активності та моніторингу показників, що сприятиме дотриманню режиму харчування та тренування 
4	Business Context (Бізнес-контекст)
4.1	Operating Environment (Робоче середовище)

Серверна частина системи для підрахунку калорій і контролю харчування буде реалізована на платформі ASP.NET Core, що дозволить створити високопродуктивне та масштабоване рішення для обробки запитів користувачів у реальному часі. Завдяки ASP.NET Core можна створити RESTful API, яке забезпечить взаємодію мобільних та веб-клієнтів із сервером, підтримуючи швидкий доступ до інформації про харчування, фізичну активність і дані IoT-пристроїв.

Для зберігання даних про користувачів, їхні щоденні цілі, історію харчування, активність та аналітику прогресу буде використовуватись Microsoft SQL Server. Цей вибір дозволяє забезпечити цілісність і надійність даних, а також легке масштабування системи для підтримки зростаючої кількості користувачів. Microsoft SQL Server надає механізми для реплікації і відновлення, що гарантує високу доступність даних і зберігання транзакцій, що підвищує стабільність платформи.

Для зберігання зображень продуктів, сканованих штрих-кодів та інших файлів буде використано Firebase Cloud Storage. Це рішення забезпечить надійне зберігання файлів із високою швидкістю доступу, що є критично важливим для підтримки функції швидкого сканування та завантаження зображень.

Обмін даними між IoT-пристроями, наприклад, інтелектуальними вагами, і сервером буде реалізовано через Azure IoT Hub, що дозволить передавати дані про вагу користувачів у режимі реального часу з мінімальними затримками.

Для розробки веб-клієнта, який буде доданий у майбутніх релізах, буде використано Flutter for Web, що забезпечить динамічний інтерфейс користувача для відстеження харчування та активності. Provider або Riverpod буде використовуватися для керування станом веб-клієнта, що дозволить синхронізувати дані між різними компонентами інтерфейсу, зберігаючи цілісність інформації. Dio буде застосований для обробки HTTP-запитів, що забезпечить стабільну взаємодію між веб-клієнтом і сервером.

Flutter також буде використано для розробки мобільного клієнта, що дозволить забезпечити зручний і швидкий доступ до платформи. Firebase Authentication надасть можливість безпечної авторизації користувачів через облікові записи Google та інших соцмереж, спрощуючи вхід для користувачів і консультантів.



Основні вимоги до робочого середовища:

Доступність: Система повинна бути доступною 24/7 завдяки реплікаціям даних у Microsoft SQL Server та Firebase Cloud Storage, що дозволить уникати збоїв у роботі платформи.

Надійність: Microsoft SQL Server гарантує цілісність даних через транзакції та механізми відновлення даних, що дозволить користувачам зберігати свої дані та відновлювати їх у разі неполадок.

Продуктивність: Поєднання ASP.NET Core з Microsoft SQL Server забезпечує високу продуктивність обробки запитів, що є необхідним для швидкої обробки даних IoT-пристроїв та оновлення харчового щоденника в режимі реального часу.

Цілісність даних: Транзакції в Microsoft SQL Server забезпечують надійність та безпеку даних, а Azure IoT Hub дозволяє стабільно передавати інформацію про фізичну активність із IoT-пристроїв на сервер.
